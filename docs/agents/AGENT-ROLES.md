# 自助爬虫平台 — Agent 角色手册

这个文档记录了项目中各专项 Agent 的职责与调用场景，方便随时追溯。

---

### 🏛️ Agent 1: 产品经理 (Product Designer)
- **ID 标识**: `crawler-prd-expert`
- **核心作用**: 负责项目的“大脑”。将模糊的需求细化为结构化的文档，定义用户交互流程、API 接口契约及数据库表结构。
- **产出成果**: 飞书云文档（PRD、API设计、UI原型）。
- **交付方式**: 每次更新后自动同步链接给用户。
- **何时找它**: 需要修改功能逻辑、增加新模块、或者纠结业务流程时。

### 💻 Agent 2: 全栈开发工程师 (Dev Engineer)
- **ID 标识**: `crawler-dev-fullstack`
- **核心作用**: 负责项目的“双手”。根据 PRD 编写 FastAPI 后端代码（异步任务、爬虫逻辑、ORM）和 Vue3 前端代码（Vite, Tailwind）。
- **产出成果**: 飞书云文档（技术架构图、API 设计规范）、`backend/`、`frontend/`、`docker-compose.yml`。
- **交付方式**: 任何代码设计与架构说明均首选飞书文档发布链接。

### 🧪 Agent 3: QA 测试与审计 (Test & Audit)
- **ID 标识**: `crawler-test-auditor`
- **核心作用**: 负责项目的“眼睛”。编写自动化测试脚本，验证爬虫对不同站点的兼容性，进行压力测试及安全审计。
- **产出成果**: 飞书云文档（测试计划、自动化测试报告）、`tests/`。
- **交付方式**: 测试结论与报告必须以飞书文档链接形式同步给用户。

### 🔍 Agent 4: 查询平台 (Query Platform Expert)
- **ID 标识**: `crawler-query-api-helper`
- **核心作用**: 负责项目的“连接器”。通过对话接收指令，自动调用内部/外部 API 接口进行数据查询、任务状态同步或系统集成操作。
- **产出成果**: 实时 API 调用结果、自动化查询日志。
- **何时找它**: 需要通过自然语言查询任务状态、触发特定 API 动作、或进行跨系统数据拉取时。
